## Problem Statement
做一个 QQ 群机器人：用户 @bot 提问 → AI 回答；定时推新闻/天气/自定义内容到配置的群列表。本地开发，云端部署。扩展性是核心需求，模型能力决定最终效果。

## What Makes This Cool
- 扩展性强：插件化架构，新增功能装插件就行
- 模型可插拔：本地 Ollama / 云端 DeepSeek 随时切换，模型质量由你决定
- 上下文支持：多轮对话记忆
- 多模态预留：图片理解、语音、生成都能加

## 插件体系

### 内置插件
- `plugins.chat` — AI 问答，@bot 触发，支持多轮上下文
- `plugins.broadcast` — 群列表管理，手动广播
- `plugins.scheduler` — 定时任务，调用内容源
- `plugins.admin` — 管理员命令（增删群、切换模型、重载配置）

### 扩展插件（按需安装）
- `plugins.rss` — RSS 订阅推送
- `plugins.weather` — 天气查询
- `plugins.image_understand` — 图片理解（多模态）
- `plugins.asr` — 语音转文字
- `plugins.image_gen` — 图片生成

### 插件开发规范
新建插件只需：
```python
# plugins/my_plugin/__init__.py
from nonebot import on_command, Plugin
from nonebot.adapters import Event

my_plugin = Plugin("my_plugin")

@on_command("my_cmd", plugins=[my_plugin]).handle()
async def handler(event: Event):
    await my_plugin.finish("response")
```

## LLM 适配层 (llm_adapter/)

**这是最核心的扩展层，所有 AI 能力从这里走。**

### 接口设计
```python
class BaseLLMAdapter(ABC):
    @abstractmethod
    async def chat(
        self,
        prompt: str,
        context: list[dict] | None = None,  # [{"role": "user"/"assistant", "content": str}]
        *,
        image: bytes | None = None,           # 多模态：图片bytes
        model: str | None = None,              # 可选：强制指定模型
        **kwargs
    ) -> str:
        """返回 AI 回复文本"""
        pass

    @abstractmethod
    async def chat_stream(
        self,
        prompt: str,
        context: list[dict] | None = None,
        *,
        image: bytes | None = None,
        **kwargs
    ) -> AsyncGenerator[str, None]:
        """流式返回（用于打字机效果）"""
        pass
```

### 已实现适配器

#### OllamaAdapter（本地优先）
```env
LLM_PROVIDER=ollama
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_MODEL=b5e600c99fb2a41d7ad4db45c1eb23b99aaf4159410009a39e5186afbb909d45
```
- 调用 `http://localhost:11434/api/chat`
- 支持流式输出
- 模型可热切换（换模型 ID 即可）
- 图片理解：ollama 的 `llava` 模型支持图片输入，接口自动兼容

#### DeepSeekAdapter（云端部署用）
```env
LLM_PROVIDER=deepseek
DEEPSEEK_API_KEY=sk-xxxx
DEEPSEEK_MODEL=deepseek-chat
```
- 调用 `https://api.deepseek.com/chat/completions`
- 支持 function calling（未来扩展工具调用）
- 图片理解：DeepSeek V2.5+ 原生多模态

### 新增 LLM 适配器
在 `llm_adapter/` 新建文件：
```python
# llm_adapter/new_provider.py
class NewProviderAdapter(BaseLLMAdapter):
    async def chat(self, prompt, context, *, image=None, **kwargs) -> str:
        # 调用新 provider 的 API
        ...
    async def chat_stream(self, ...):
        ...
```
然后在 `llm_adapter/__init__.py` 注册即可。

## 上下文扩展

### 当前设计（基础多轮）
```python
# 每次对话存 Redis/JSON
SESSION_KEY = "chat:history:{user_id}"
# [{role: "user", content: "..."}, {role: "assistant", content: "..."}]

# LLM 调用时组装
messages = history[-10:]  # 最近10轮
messages.append({"role": "user", "content": prompt})
```

### 后续扩展方向
| 扩展 | 怎么做 | 改动范围 |
|------|--------|----------|
| 语义向量检索 | 把历史对话向量化，检索相关内容组装进 prompt | 新增 `plugins/memory/` |
| 知识库关联 | RAG，把用户历史行为作为 context 注入 | 新增 `llm_adapter.rag()` |
| 长期记忆 | 定期摘要压缩，把长对话精华提炼出来 | `plugins/memory/summarizer.py` |

## 多模态扩展

### 图片理解
1. 换模型：Ollama 用 `llava`，DeepSeek 用 V2.5+
2. `BaseLLMAdapter.chat()` 的 `image` 参数传图片 bytes
3. `plugins/chat/` 收到图片消息时自动触发

```python
# 收到图片消息时
image_bytes = await event.get_image()
response = await llm.chat(prompt, image=image_bytes)
```

### 语音理解（ASR）
```python
# 收到语音消息时
audio_bytes = await event.get_voice()
text = await asr_service.transcribe(audio_bytes)  # Whisper API
response = await llm.chat(text)
```

### 图片生成
```python
# 用户说 "画一个..."
prompt = extract_image_prompt(user_message)
image_url = await image_gen.generate(prompt)  # DALL-E / Stable Diffusion
await bot.send(MessageSegment.image(image_url))
```

## 开发顺序

### Phase 1: 跑通基础（1-2天）
1. NoneBot2 项目初始化 (`nb create`)
2. LLOneBot / LLOneBot 配置，连上 QQ
3. 确认收消息、发消息走通

### Phase 2: AI 问答（2-3天）
4. LLM 适配层（先 Ollama）
5. chat 插件，@bot 回复
6. 对话历史（JSON 文件存）

### Phase 3: 广播 + 定时（1-2天）
7. broadcast 插件，群列表管理
8. scheduler 插件 + 一个内容源
9. 定时广播跑通

### Phase 4: 云端部署（1天）
10. Docker compose 打包
11. 部署到云，切 DeepSeek API

### 后续扩展（按需）
- 图片理解 → 换 vision 模型，加 `plugins/ext/image_understand/`
- ASR → 加 `plugins/ext/asr/`
- 知识库 RAG → 加 `llm_adapter/rag.py`
- 长期记忆 → 加 `plugins/memory/summarizer.py`

# plugins/scheduler/sources/base.py
from abc import ABC, abstractmethod

class BaseSource(ABC):
    name: str  # 唯一标识，如 "news"

    @abstractmethod

        """返回要广播的内容文本"""
