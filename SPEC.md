## 1. 项目概述

### 1.1 项目名称
`qq-bot` — 基于 NoneBot2 的 QQ 群机器人

### 1.2 项目目标
- 用户 `@bot` 提问 → AI 回答
- 定时推送新闻/天气/自定义内容到配置的群列表
- 本地开发，云端部署
- 扩展性为核心需求

### 1.3 技术栈
| 层级 | 技术 |
|------|------|
| 消息协议 | QQ (LLOneBot / OneBot V11) |
| 核心框架 | NoneBot2 (Python 3.10+) |
| LLM 适配层 | LongCat（文字+图片理解）/ Pollinations（图片生成） |
| 定时任务 | nonebot-plugin-scheduler / APScheduler |
| 会话存储 | JSON 文件 (可选 Redis) |
| 部署 | Docker + Docker Compose |

### 3.1 LLM 适配器 (`llm_adapter/`)

#### 3.1.1 基类定义

```python
# llm_adapter/base.py
from abc import ABC, abstractmethod
from typing import AsyncGenerator

class BaseLLMAdapter(ABC):
    @abstractmethod
    async def chat(
        self,
        prompt: str,
        context: list[dict] | None = None,
        *,
        image: bytes | None = None,
        model: str | None = None,
        **kwargs
    ) -> str:
        """
        返回 AI 回复文本

        Args:
            prompt: 用户输入的提示词
            context: 对话历史上下文，格式为 [{"role": "user"|"assistant", "content": str}]
            image: 可选，图片字节数据（用于多模态）
            model: 可选，强制指定模型
            **kwargs: 其他provider-specific参数

        Returns:
            str: AI 回复文本

        Raises:
            LLMError: 调用失败时抛出
        """
        pass

    @abstractmethod
    async def chat_stream(
        self,
        prompt: str,
        context: list[dict] | None = None,
        *,
        image: bytes | None = None,
        **kwargs
    ) -> AsyncGenerator[str, None]:
        """
        流式返回（用于打字机效果）

        Args:
            prompt: 用户输入的提示词
            context: 对话历史上下文
            image: 可选，图片字节数据
            **kwargs: 其他provider-specific参数

        Yields:
            str: 逐步返回的文本片段

        Raises:
            LLMError: 调用失败时抛出
        """
        pass

    @property
    @abstractmethod
    def provider_name(self) -> str:
        """返回适配器名称，如 'ollama', 'deepseek'"""
        pass
```

#### 3.1.2 OllamaAdapter

```python
# llm_adapter/ollama.py
class OllamaAdapter(BaseLLMAdapter):
    def __init__(
        self,
        base_url: str = "http://localhost:11434",
        model: str | None = None,
        timeout: int = 120
    ):
        """
        Args:
            base_url: Ollama 服务地址
            model: 模型名称，默认使用环境变量 OLLAMA_MODEL
            timeout: 请求超时时间（秒）
        """
        self.base_url = base_url.rstrip("/")
        self.model = model
        self.timeout = timeout

    async def chat(
        self,
        prompt: str,
        context: list[dict] | None = None,
        *,
        image: bytes | None = None,
        model: str | None = None,
        **kwargs
    ) -> str:
        """
        调用 Ollama /api/chat 接口

        Args:
            prompt: 提示词
            context: 对话历史
            image: base64 编码的图片（llava 模型支持）
            model: 模型名称，覆盖默认

        Returns:
            str: AI 回复

        Raises:
            LLMConnectionError: 无法连接到 Ollama
            LLMResponseError: Ollama 返回错误
        """
        ...

    async def chat_stream(self, prompt, context, *, image=None, **kwargs) -> AsyncGenerator[str, None]:
        """流式调用 /api/chat"""
        ...

    @property
    def provider_name(self) -> str:
        return "ollama"
```

#### 3.1.4 PollinationsAdapter（图片生成）

```python
# llm_adapter/pollinations.py
async def generate_image(prompt: str, width: int = 1024, height: int = 1024) -> str:
    """
    调用 Pollinations AI 生成图片，返回图片 URL。
    Pollinations 完全免费，无需 API key。
    """
    encoded_prompt = quote(prompt)
    return (
        f"https://image.pollinations.ai/"
        f"?prompt={encoded_prompt}"
        f"&width={width}"
        f"&height={height}"
        f"&nologo=true"
    )
```

**说明：** Pollinations 是免费开源的 AI 生图服务，无需注册 API key，直接通过 URL 方式调用。

#### 3.1.5 适配器工厂

```python
# llm_adapter/__init__.py
from llm_adapter.base import BaseLLMAdapter
from llm_adapter.ollama import OllamaAdapter
from llm_adapter.deepseek import DeepSeekAdapter
from llm_adapter.openai import OpenAIAdapter

_LLM_ADAPTERS: dict[str, type[BaseLLMAdapter]] = {
    "longcat": LongCatAdapter,
}

def get_adapter(provider: str | None = None) -> BaseLLMAdapter:
    """
    根据 provider 名称获取适配器实例

    Args:
        provider: 提供商名称，默认从 LLM_PROVIDER 环境变量读取

    Returns:
        BaseLLMAdapter: 适配器实例

    Raises:
        ValueError: provider 不支持
        EnvironmentError: 缺少必要的环境变量
    """
    if provider is None:
        provider = os.getenv("LLM_PROVIDER", "ollama")

    adapter_class = _LLM_ADAPTERS.get(provider)
    if not adapter_class:
        raise ValueError(f"Unsupported LLM provider: {provider}. Supported: {list(_LLM_ADAPTERS.keys())}")

    return adapter_class()

def register_adapter(name: str, adapter_class: type[BaseLLMAdapter]) -> None:
    """注册新的 LLM 适配器"""
    _LLM_ADAPTERS[name] = adapter_class
```

class BaseSource(ABC):

    @abstractmethod
    async def fetch(self) -> str:
        """
        获取内容

## 8. API 接口

### 8.1 OneBot API (内部)

NoneBot2 通过 LLOneBot 与 QQ 服务器通信，使用 OneBot V11 协议。

**主要事件：**
| 事件类型 | 说明 |
|----------|------|
| `message.group` | 群消息 |
| `message.private` | 私聊消息 |
| `notice.group_increase` | 群成员增加 |
| `notice.group_decrease` | 群成员减少 |
| `request.friend` | 好友请求 |
| `request.group` | 加群请求 |

**主要 API：**
| API | 说明 |
|-----|------|
| `send_msg` | 发送消息 |
| `send_group_msg` | 发送群消息 |
| `send_private_msg` | 发送私聊消息 |
| `get_group_member_list` | 获取群成员列表 |
| `get_friend_list` | 获取好友列表 |

### 8.2 管理员 HTTP API (可选扩展)

如需提供 HTTP 管理接口，可扩展以下端点：

| 端点 | 方法 | 说明 |
|------|------|------|

| `/api/broadcast` | POST | 手动触发广播 |

## 10. 错误码

### 10.1 LLM 错误

| 错误码 | 名称 | 说明 |
|--------|------|------|
| `LLM001` | `LLMConnectionError` | 无法连接到 LLM 服务 |
| `LLM002` | `LLMResponseError` | LLM 返回响应格式错误 |
| `LLM003` | `LLMTimeoutError` | LLM 请求超时 |
| `LLM004` | `LLMAuthenticationError` | LLM API 认证失败 |

### 10.2 内容源错误

| 错误码 | 名称 | 说明 |
|--------|------|------|
| `SRC001` | `SourceConnectionError` | 无法连接到内容源 |
| `SRC002` | `SourceParseError` | 内容解析失败 |
| `SRC003` | `SourceRateLimitError` | 内容源请求频率超限 |

### 10.3 插件错误

| 错误码 | 名称 | 说明 |
|--------|------|------|
| `PLG001` | `PluginNotFoundError` | 插件不存在 |
| `PLG002` | `PluginLoadError` | 插件加载失败 |
| `PLG003` | `PluginPermissionError` | 权限不足 |

### 10.4 错误处理规范

```python
# 所有模块错误应继承自 BaseError
class BaseError(Exception):
    code: str
    message: str

# 示例
class LLMConnectionError(BaseError):
    code = "LLM001"
    message = "无法连接到 LLM 服务"
```

## 13. 开发顺序

### Phase 1: 跑通基础（1-2天）
1. NoneBot2 项目初始化 (`nb create`)
2. LLOneBot / LLOneBot 配置，连上 QQ
3. 确认收消息、发消息走通

### Phase 2: AI 问答（2-3天）
4. LLM 适配层（先 Ollama）
5. chat 插件，@bot 回复
6. 对话历史（JSON 文件存）

### Phase 3: 广播 + 定时（1-2天）
7. broadcast 插件，群列表管理
8. scheduler 插件 + 一个内容源
9. 定时广播跑通

### Phase 4: 云端部署（1天）
10. Docker compose 打包
11. 部署到云

### 后续扩展（按需）
- 图片编辑 → 引用图片 + 修改描述，通过 LongCat 图片理解 + Pollinations 生图
- 知识库 RAG → 加 `llm_adapter/rag.py`
- 长期记忆 → 加 `plugins/memory/summarizer.py`
